# Zero_Nine 项目架构与使用说明

## 一、项目概述

**Zero_Nine** 是一个以 Rust 编写的统一编排内核，将四个上游项目的核心能力整合为一个可落地的工程系统：

| 上游项目 | Zero_Nine 映射层 | 职责 |
|---------|----------------|------|
| **OpenSpec** | 需求层 (`zn-spec`) | 需求、设计与任务工件管理 |
| **Superpowers** | 执行层 (`zn-exec`) | 结构化执行、质量约束与验证 |
| **Ralph-loop** | 调度层 (`zn-loop`) | 长时循环、状态恢复与任务推进 |
| **OpenSpace** | 进化层 (`zn-evolve`) | 技能评估、演化候选与持续优化 |

## 四、核心数据模型 (zn-types)

### 4.1 宿主类型
```rust
enum HostKind {
    ClaudeCode,    // Claude Code CLI
    OpenCode,      // OpenCode CLI
    Terminal,      // 纯终端模式
}
```

### 4.2 提案状态
```rust
enum ProposalStatus {
    Draft,      // 草稿
    Ready,      // 就绪可执行
    Running,    // 执行中
    Completed,  // 已完成
    Archived,   // 已归档
}
```

### 4.3 任务状态
```rust
enum TaskStatus {
    Pending,    // 待处理
    Running,    // 执行中
    Completed,  // 已完成
    Failed,     // 失败
    Blocked,    // 被阻塞
}
```

### 4.4 循环阶段
```rust
enum LoopStage {
    Idle,           // 空闲
    SpecDrafting,   // 规格起草
    Ready,          // 就绪
    RunningTask,    // 任务执行
    Verifying,      // 验证中
    Retrying,       // 重试中
    Escalated,      // 已升级
    Archived,       // 已归档
}
```

### 4.5 核心数据结构

| 结构体 | 作用 |
|-------|------|
| `ProjectManifest` | 项目级配置（版本、宿主、策略） |
| `Proposal` | 需求提案（目标、状态、任务列表） |
| `TaskItem` | 任务项（ID、标题、依赖、契约） |
| `TaskGraph` | 任务 DAG（任务 + 依赖边） |
| `LoopState` | 循环状态（当前任务、迭代次数、阶段） |
| `ExecutionReport` | 执行报告（成功/失败、证据、 verdict） |
| `SkillEvaluation` | 技能评分（延迟、成本、分数） |
| `EvolutionCandidate` | 演化候选（源技能、补丁、置信度） |
| `BrainstormSession` | 头脑风暴会话（问题、答案、裁决） |

## 五、模块职责与互动关系

### 5.1 zn-types（共享类型）
- **职责**: 定义所有模块共享的数据模型
- **依赖**: 无
- **被依赖**: 所有其他 crates

### 5.2 zn-spec（需求层）
- **职责**: 
  - 管理 `.zero_nine/` 工件目录布局
  - 创建/保存/加载 Proposal
  - 管理 Brainstorm 会话
  - 生成任务 DAG 和进度文件
  - 规格验证报告
- **关键函数**:
  - `create_proposal()`: 创建新提案
  - `create_proposal_from_brainstorm()`: 从头脑风暴创建提案
  - `validate_proposal_spec()`: 验证规格完整性
  - `update_progress_markdown()`: 更新进度

### 5.3 zn-exec（执行层）
- **职责**:
  - 任务分类（Brainstorming/Planning/Implementation/Verification）
  - 构建执行计划和信封
  - 工作空间准备（In-Place/GitWorktree/Sandboxed）
  - 执行验证动作
  - 分支完成自动化
- **关键函数**:
  - `build_plan()`: 构建执行计划
  - `prepare_workspace()`: 准备工作空间
  - `execute_plan()`: 执行计划
  - `finish_branch()`: 完成分支（Merge/PR）

### 5.4 zn-loop（调度层）
- **职责**:
  - 项目初始化
  - 头脑风暴循环
  - 目标执行驱动
  - 状态恢复
  - 批量任务调度
- **关键函数**:
  - `run_goal()`: 运行目标
  - `resume()`: 从断点恢复
  - `execute_proposal()`: 执行提案
  - `choose_next_ready_batch()`: 选择可执行任务批

### 5.5 zn-evolve（进化层）
- **职责**:
  - 执行结果评分
  - 生成演化候选
- **关键函数**:
  - `evaluate()`: 评分（0.33-0.97）
  - `propose_candidate()`: 生成候选补丁

### 5.6 zn-host（宿主适配）
- **职责**:
  - 宿主检测
  - 导出适配文件
- **关键函数**:
  - `detect_host()`: 检测宿主类型
  - `export_adapter_files()`: 导出命令/技能文件

## 七、详细启动和使用步骤

### 7.1 环境准备

**前提条件**:
- Rust 环境（最新 stable 工具链）
- Git（用于 worktree 功能）
- 可选：GitHub CLI（用于 PR 创建）

### 7.2 编译项目

```bash
cd /Users/tangxingzhuang/Freedom/Zero_Nine
cargo build
```

编译成功后，可执行文件名为 `zero-nine`。

### 7.3 验证安装

```bash
cargo run -p zn-cli -- --help
```

### 7.4 初始化项目

在目标项目目录初始化 Zero_Nine：

```bash
# 对于 Claude Code 宿主
zero-nine init --project /path/to/your/repo --host claude-code

# 对于 OpenCode 宿主
zero-nine init --project /path/to/your/repo --host opencode

# 纯终端模式
zero-nine init --project /path/to/your/repo --host terminal
```

这会创建：
- `.zero_nine/` 工作目录
- `.zero_nine/manifest.json` 项目配置
- 所有必要的子目录（proposals/, loop/, runtime/, evolve/, specs/）

### 7.5 运行一句话目标

核心功能：用一句话目标启动完整四层流程

```bash
zero-nine run --project /path/to/your/repo --host claude-code --goal "为当前仓库增加单元测试"
```

**执行流程**:
1. **Brainstorming**: 生成澄清问题，等待用户回答
2. **Spec Capture**: 创建 Proposal 和 OpenSpec 工件
3. **Execution**: 调度任务、执行、验证
4. **Evolution**: 评分、生成候选

### 7.6 查看状态

```bash
zero-nine status --project /path/to/your/repo
```

输出示例：
```
manifest: present
brainstorm_session: bs-xxx
brainstorm_verdict: Ready
proposal: 20260408-feature-name
goal: 为当前仓库增加单元测试
status: Running
tasks: 2/4 completed
scheduler: 1 runnable / 1 blocked / 1 pending
runnable_tasks: 3:执行 guarded 实现
blocked_details: 4:验证等待依赖
loop_stage: RunningTask
iteration: 3
current_task: 3
```

### 7.7 恢复执行

如果执行中断，可以恢复：

```bash
zero-nine resume --project /path/to/your/repo --host claude-code
```

### 7.8 导出宿主适配文件

```bash
zero-nine export --project /path/to/your/repo
```

这会生成：
- `.claude/commands/zero-nine.md`
- `.claude/skills/zero-nine-orchestrator/SKILL.md`
- `.opencode/commands/zero-nine.md`
- `.opencode/skills/zero-nine-orchestrator/SKILL.md`

### 7.9 宿主页使用

#### Claude Code

在 Claude Code 中，使用斜杠命令：

```
/zero-nine 为当前仓库增加单元测试
```

如果 Brainstorming 未 Ready，继续用同一命令回答问题：

```
/zero-nine 我的问题是...（答案）
```

#### OpenCode

在 OpenCode 中，同样使用：

```
/zero-nine 为当前仓库增加单元测试
```

## 九、任务调度逻辑

### 9.1 DAG 调度

任务执行遵循依赖:
1. 只有依赖全部完成的任务才可执行
2. 支持并行执行（默认 2 个并行窗口）
3. 失败任务可重试（默认 2 次）

### 9.2 资源约束

| 资源类型 | 默认限制 |
|---------|---------|
| 并行窗口 | 2 任务 |
| Worktree 槽位 | 2（有 finish_branch 时为 1） |
| Finish 槽位 | 1 |

### 9.3 重试策略

```
Pending → Running → Failed → Retrying → Running → ...
                              │
                              └─(超过 max_retries)─→ Escalated
```

## 十、执行模式链

## 十一、验证动作

执行层支持自动验证动作：

| 动作名 | 命令 | 必需 |
|-------|------|------|
| tests | 自动检测项目测试命令 | 是 |
| review | `git diff --stat && git diff --check` | 是 |

### 测试命令自动检测

| 项目类型 | 检测文件 | 测试命令 |
|---------|---------|---------|
| Rust | Cargo.toml | `cargo test --all-targets` |
| Python | pyproject.toml/requirements.txt | `pytest -q` |
| Node.js | package.json | `npm test`/`yarn test`/`pnpm test` |

## 十二、分支完成自动化

支持四种完成动作：

| 动作 | 描述 |
|-----|------|
| Merge | 合并到当前分支 |
| PullRequest | 创建 GitHub PR |
| Discard | 丢弃分支和 worktree |
| Keep | 保留分支继续迭代 |

**前提条件**:
- 需要显式确认（`allow_remote_finish=true`）
- 工作树必须干净
- GitHub CLI 已认证（PR 模式）

## 十五、常见问题

### Q1: Brainstorming 如何判定 Ready？
- 所有澄清问题已回答
- 答案不是低信号（少于 3 词）
- 规格验证通过

### Q2: 任务失败后如何处理？
1. 自动重试（最多 2 次）
2. 超过重试次数后 Escalated
3. 生成演化候选补丁
4. 等待人工干预

### Q3: 如何查看子代理执行记录？
```bash
cat .zero_nine/runtime/subagents/task-1/subagent-recovery-ledger.json
```

### Q4: Git worktree 创建失败？
如果仓库没有初始 commit，会自动降级为 In-Place 模式。
